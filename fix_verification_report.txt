问题 1: 多栏PDF标题被拆分
--------------------------------------------------
问题描述: 多栏PDF处理把整页固定二分为两栏，跨栏标题被强行劈成左右两栏后按列读出，
         sample2标题断成"基于深度学习的图像识别技 / 术研究"两行

修复分析:
---------
已修复 - 新增智能行检测和跨栏元素识别算法

关键代码位置: app/pdf_utils/extractor.py

1. 新增 Line 类 (第78-96行):
   - 用于聚合同一行的文本片段
   - 计算行的宽度、高度、中心位置、最大字号等属性
   - 为跨栏判定提供基础

2. 新增 _detect_lines() 方法 (第331-361行):
   - 按y坐标聚类文本片段，自动识别文本行
   - 使用行高的40%作为容差，判断是否属于同一行
   - 解决标题文字被拆分到不同行的问题

3. 改进 _detect_columns() 方法 (第363-494行):
   - 不再固定二分页面，而是基于内容智能检测
   - 跨栏判定条件 (第386-396行):
     * 行宽度 > 75% 页面宽度，判定为跨栏
     * 字号 > 14，判定为跨栏（大标题）
   - 列检测使用直方图峰值分析 (第412-459行):
     * 统计所有文本片段的x0坐标直方图
     * 检测峰值确定列中心
     * 自动确定实际列数（支持2栏、3栏等）
   - 备选方案：检测x坐标间隙确定列分隔线 (第461-488行)

4. 改进 _sort_by_reading_order() 方法 (第552-596行):
   - 跨栏元素 (is_span_column=True) 优先排列
   - 正文按列优先排序（先读完左栏，再读右栏）
   - 确保标题不会被错误插入到正文中间

修复效果预期:
- sample2的大标题"基于深度学习的图像识别技术研究"会被正确识别为跨栏元素
- 不会被拆分为"基于深度学习的图像识别技 / 术研究"
- 阅读顺序：标题 → 左栏全文 → 右栏全文

问题 2: 多个独立表格被合并识别
--------------------------------------------------
问题描述: sample3含两张独立表格只识别出一个表，第二张被合进同一识别结果

修复分析:
---------
已修复 - 双模式识别 + 间距分离算法

关键代码位置: app/pdf_utils/extractor.py

1. 改进 _extract_page_tables() 方法 (第712-768行):
   - 同时使用 camelot 的两种识别模式:
     * lattice 模式 (第715行): 适用于有框线的表格
     * stream 模式 (第715行): 适用于无框线的流式表格
   - 两种模式结果合并，提高识别率

2. 新增 _merge_overlapping_tables() 方法 (第770-790行):
   - 分离条件 (第779-784行):
     * 垂直间距 > 20 像素 → 判定为独立表格
     * 水平重叠 < 50% → 判定为独立表格
   - 只有当两个表格垂直靠近且水平大部分重叠时才会合并

3. 新增 _fallback_table_detection() 方法 (第792-903行):
   - 当 camelot 识别失败时的备选方案
   - 基于文本布局分析检测表格:
     * 检测具有规则x坐标间隔的多行文本
     * 每行至少2个元素，元素间有明显间隙
     * 根据行间距和宽度相似度分组表格
   - 支持检测无框线的纯文本表格

修复效果预期:
- sample3中的两张独立表格会被正确识别为2个（或更多）独立表格
- 表格之间有足够间距时不会被错误合并
- 同时支持有框线和无框线表格的识别

--------------------------------------------------
问题 3: 扫描PDF无OCR通路
--------------------------------------------------
问题描述: 扫描转PDF形态在代码中无OCR通路，pdf2image与opencv-python列在依赖里
         却未在extractor中实际使用

修复分析:
---------
已修复 - 完整的OCR处理通路，自动检测扫描页并调用OCR

关键代码位置: app/pdf_utils/extractor.py

1. 新增依赖导入 (第1-11行):
   - from PIL import Image, ImageFilter
   - import cv2 (opencv-python)
   - from pdf2image import convert_from_bytes
   - 两个依赖库已实际使用

2. 新增 _is_scanned_page() 方法 (第116-138行):
   - 自动检测页面是否为扫描件:
     * 提取的文本长度 < 10 字符
     * 且文本块中无有效文本内容
   - 结果缓存，避免重复检测

3. 新增 _extract_ocr() 方法 (第140-243行):
   - 完整的OCR处理流程:
     a. PDF转图像 (第142-147行):
        - 使用 pdf2image.convert_from_bytes
        - DPI=300，保证识别精度
     
     b. OpenCV预处理 (第152-159行):
        - 灰度转换: cv2.cvtColor
        - 二值化: cv2.threshold (OTSU算法)
        - 形态学操作: cv2.morphologyEx (开运算 + 膨胀)
        - 去除噪声，增强文字轮廓
     
     c. OCR识别（双引擎支持）:
        - 优先使用 pytesseract (第161-205行):
          * 支持中英文混合识别 (lang='chi_sim+eng')
          * 获取每个文字块的位置、大小、置信度
          * 置信度过滤 (>60)
          * 坐标从图像空间映射到PDF空间
        - 回退到 PyMuPDF OCR (第207-239行):
          * 使用 page.get_textpage_ocr()
          * pytesseract 未安装时自动回退

4. 新增 _extract_page_layout() 调度方法 (第620-629行):
   - 智能调度流程:
     1. 检测是否为扫描页 → 是则调用 OCR
     2. 尝试原生文本提取 → 失败则回退到 OCR
   - 对用户透明，自动选择最佳提取方式

5. 元信息标记 (第251, 265行):
   - PDFMetadata.is_scanned 字段标记文档是否含扫描页
   - 方便调用方了解文档类型

修复效果预期:
- sample5（扫描PDF）会被自动检测为扫描文档
- pdf2image 和 opencv-python 会被实际调用处理图像
- OCR引擎会提取扫描页中的文本内容
- 即使原生文本为空，也能通过OCR获取内容
