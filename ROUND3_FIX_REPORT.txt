问题 1: sample2多栏标题仍断成两段且阅读顺序混乱
======================================================================

问题描述:
-----------
sample2多栏标题仍断成两段，正文段与标题严重交叉混排，
阅读顺序比修复前更乱。

根本原因分析:
-----------

### 1. 行检测的来源不可靠：
- pdfplumber提取的是词级片段，每个词独立，然后我们自己聚合成行
- 自行实现的行检测算法容易出错，特别是大字体标题
- 大标题的y坐标有细微差异时很容易被拆成多行

### 2. 阅读顺序排序逻辑过于复杂：
- 前一版本的_sort_by_reading_order使用了复杂的块索引和穿插逻辑
- 逻辑复杂，容易出现bug，导致正文与标题交叉混排

### 3. 跨栏判定基于有问题：
- 行聚合不正确导致跨栏判定也会受影响

修复方案:
-----------

### 1. 从根源解决 - 优先使用PyMuPDF的行检测

### 3. 重写阅读顺序排序，最简单直接的逻辑

```python
        span_in_line.sort(key=lambda f: f.x0)
```

问题 2: 表格识别从过识别跳到漏识别另一极
======================================================================

问题描述:
-----------
表格识别在两轮修复中从过识别跳到漏识别另一极：
- sample3含两张独立表只识别一张
- 文档型sample4只识1张表（目录被误识别）
- 识别精度大幅倒退

根本原因分析:
-----------

### 1. 阈值一刀切：
- 前一版本对lattice和stream模式使用了相同的严格阈值
- lattice模式（有框线）其实很可靠，不应该用太严格的阈值
- stream模式（无框线）容易误识别，需要严格阈值

### 2. 缺少目录/列表类结构化文本过滤：
- 目录、列表等文本有规则间距的文本很容易被stream模式误识别
- 没有针对性的过滤逻辑

### 3. 缺少表格分离机制：
- camelot可能把同一页的两张间距较小的表格合并识别
- 没有分离合并表格的机制

修复方案:
-----------

### 2.1 双模式差异化阈值策略
文件: app/pdf_utils/extractor.py:790-801

| 模式      | accuracy | 最小行列 | 非空单元格比例 |
|-----------|----------|----------|--------------|
| lattice   | >=40     | 2x2      | >=20%        |
| stream    | >=90     | 4x3      | >=60%        |

关键代码:
```python
if flavor == "lattice":
    if ct.accuracy < 40:
        continue
    if rows < 2 or cols < 2:
        continue
    non_empty_ratio = 0.2
else:
    if ct.accuracy < 90:
        continue
    if rows < 4 or cols < 3:
        continue
    non_empty_ratio = 0.6
```

### 2.2 新增目录/列表类结构化文本检测
文件: app/pdf_utils/extractor.py:845-895

新增is_likely_toc检测逻辑：

1. **章节号检测:
```python
for t in first_col_texts:
    if re.match(r'^[\d\.]+$', t):  # 1.2.3
```

2. **页码检测（最后一列是1-500数字）:
```python

```

3. **省略号检测（目录常用.......:
```python
if re.search(r'[.．·]{2,}', row_text):
    has_dots += 1
```

4. **短单元格检测:
```python

    is_likely_toc = True
```

最终过滤条件（满足任一条件即判定为目录）:
- 只要: toc_number_pattern >= 40%行
或 numeric_last_col >= 40%行
或 has_dots >= 30%行
或 短单元格>60%且平均长度<8

### 2.3 新增智能表格分离算法

检测被camelot合并的独立表格：

1. 计算各行间距：
```python
for r in range(table.rows - 1):
```

2. 检测分离点：
```python
if row_heights[r] > avg_height * 1.8 and row_heights[r] > 10:

```

3. 按分离点拆分表格：
```python
new_rows = split_point - prev_split
new_data = table.data[prev_split:split_point]
new_cells = table.cells[prev_split:split_point]
```
